A Brief Description

In this Project we have designed a car parking system that will allow parking services for some authorised cars only. The authorisation will be checked by ar RFID module. And we have two PIR sensors on both sides of the gate which will detect if the car is coming in or going out.





Required Equipments

  • ATmega32 microcontroller

  • RFID module and a few RFID cards

  • PIR sensors

  • servo motor

  • LCD module

  • LED bulbs





How it works?

In this project we have designed a system that will help us with secured car parking. Also, when the car comes in front of the gate, the gate will open automatically sensing the existence of a car. We have used PIR sensors for this purpose. When a car comes, it needs to punch a card to the RFID module. If it is authorised, then it will get green signal to move forward. When it comes infront of the gate, to the PIR sensor precisely, the gate will open. When it passes through the gate and crosses the other sensor inside also, the gate will close down. If the car is not recognized by the RFID module, then the gate will not open at any case. In case of going out, PIR sensor detects a car inside the gate, and it opens the gate to let the car pass through. Similarly when it crosses the sensor outside, the gate closes. We will keep the gates open for a maximum of 15 seconds. If no action takes place after openning the gate within 15 seconds, the sytem will be reset, and the gate will be closed, allowing nobody to pass.

Algorithm



1. Start
.....2. Both PIR, RFID and MOTOR set to initial condition
.....3. LCD shows the current car count
.....4. System waits for a signal
..........5. System checks Elapsed Time
...............6. If Elapsed Time crossed limit and Any one PIR sensed a car
....................7. If the gate is open
.........................8. Close the gate
.........................9. All sensors set to initial condition
..........10. If car number is below capacity
...............11. If a card is punched
....................12. Check authorization
..........13. Else Show "Capacity Full" massage on LCD
..........14. If outer PIR senses a car
...............15. If inner PIR already sensed that car
....................16. Close the gate
....................17. Decrease count by one and update on LCD
....................18. Set both PIR to initial condition
...............19. Else if the car is authorized
....................20. Open the gate
..........21. If inner PIR senses a car
...............22. If outer PIR already sensed that car
....................23. Close the gate
....................24. Increase count by one and update on LCD
....................25. Set both PIR to initial condition
...............26. Else open the gate
..........27. Go to (4)
28. End

Problems Faced

  1. The most diffiult problem we had to solve is to manage the data received through the RFID module. At times it was giving inconsistent results.

  2. We were facing problems to maintain the sensitivity of PIR sensors. It was sensing the slightest of movements like a finger, air from electric fans, a light turning on.

  3. We faced problems with the LCD. Sometimes it was showing garbage characters, even without any change in circuits or codes.

  4. Power source management was a problem we faced. We used two arduinos, only for supplying 5V power. The servo motor, as well as the RFID module required direct 5V power, without any parallel connection. We used our AVR burners to power up the other circuits.
